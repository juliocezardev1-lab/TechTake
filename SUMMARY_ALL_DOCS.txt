7. GUIA_DEPLOY_NETLIFY.md
- Guia completo para deploy do projeto com frontend no Netlify e backend separado.
- Passo 1: Deploy do backend em Render (recomendado) ou Railway.
  - Configurar variáveis: NODE_ENV=production, PORT=3000, FRONTEND_ORIGIN=https://seu-site-netlify.netlify.app.
  - Copiar URL da API remota.
- Passo 2: Preparar backend para produção em `Back/server.js`.
  - Usar `FRONTEND_ORIGIN` e servir frontend estático apenas em produção.
- Passo 3: Atualizar frontend para usar API remota em `Front/config.js`.
- Passo 4: Deploy do frontend no Netlify com `Front` como diretório.
- Passo 5: Executar `node seed.js` uma única vez após backend online.
- Contém solução de problemas para CORS e backend inacessível.

Como executar agora que o front no Netlify e o backend no Render deram certo

1. No servidor backend (Render):
- O backend já está online e rodando no Render.
- O banco de dados SQLite deve ter sido populado com `cd Back && node seed.js`.
- Se ainda não foi feito, execute no diretório `Back`:
  - `node seed.js`

2. No frontend (Netlify):
- Certifique-se de que `Front/config.js` aponta para a URL remota do backend Render:
  - `const PROD_API_BASE_URL = "https://sua-api-no-render.onrender.com";`
- O site no Netlify deve fazer chamadas para essa URL remota.

3. Teste o fluxo completo:
- Acesse seu site Netlify e tente usar o formulário de pedidos ou login.
- Se o frontend estiver no Netlify e o backend no Render, o fluxo deve funcionar normalmente.

4. Para executar localmente novamente (se quiser desenvolver ou testar antes de redeploy):
- Abra terminal no diretório raiz do projeto.
- Execute:
  1. `cd Back`
  2. `node seed.js` (apenas uma vez para popular o banco)
  3. `npm start`
- Depois acesse localmente:
  - Cliente: `http://localhost:3000/servicos`
  - Admin: `http://localhost:3000/admin-login`
